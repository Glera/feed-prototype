# feed-prototype

Прототип приложения-ленты мини-игр (Instagram Reels / TikTok style). Лента
вертикально листается между мини-играми; каждая мини-игра — это плейбл
(self-contained HTML ≤ 5 MB). Стек: Vanilla TS + Vite (как у `playables/`).

## Запуск

```bash
cd feed-prototype
npm install
npm run dev        # http://localhost:5173
npm run build      # tsc + vite build → dist/
```

## Ключевая идея: разведение жестов

Мини-игре нужны собственные свайпы (внутри механики), а ленте — свайпы
перелистывания. Чтобы они не конфликтовали:

```
┌───────────────────────────┐
│                           │
│      GAME CONTAINER       │  ← мини-игра. Все её жесты живут здесь.
│   (.game, не на весь      │     Лента сюда НЕ подключена → не листается.
│    экран)                 │
│                           │
├───────────────────────────┤
│ ▲ Swipe to switch game ▼  │  ← .gutter — «жёлоб». Единственная зона,
└───────────────────────────┘     которая листает ленту в ОБЕ стороны
                                   (вверх → след., вниз → пред.).
```

Pointer-листенеры пейджинга висят **только** на `.gutter`. Жесты в области
игры физически не могут пролистнуть ленту. В плейсхолдере это видно: внутри
игры можно таскать «пак» 🎮 — лента при этом стоит.

## Архитектура

| Файл | Роль |
|---|---|
| `src/main.ts` | Точка входа — собирает `Feed`. |
| `src/feed.ts` | Вертикальный пейджер: drag по жёлобу, snap по дистанции + скорости флика, rubber-band на краях, ресайз. |
| `src/game-placeholder.ts` | Плейсхолдер мини-игры с независимым in-game жестом (драг пака). |
| `src/mechanics.ts` | Конфиг плейсхолдеров (намеренно почти одинаковые — тестируем свайп, не контент). |
| `src/styles.css` | Лейаут, жёлоб, frame телефона на десктопе. |

Параметры свайпа в `feed.ts`: `DISTANCE_SNAP_FRAC`, `VELOCITY_SNAP`,
`EDGE_RESISTANCE`. Высота жёлоба — `--gutter-h` в `styles.css`.

## Подгрузка мини-игр (Фаза 2)

Механики ленты — это реальные плейблы (`src/playables.ts`), загружаются в
`<iframe>` по относительному URL с того же Render-сайта (`./<id>.html`).
Плейблы тяжёлые (3–5 MB, живая физика/WebGL), поэтому смонтировано только
скользящее окно:

- `RESERVE_AHEAD = 5` — всегда держим 5 игр загруженными **впереди** текущей
  («запас»), больше не грузим.
- `KEEP_BEHIND = 1` — одна позади для мгновенного свайпа назад.
- Остальные iframe'ы выгружаются (`unmount`) → освобождают память/CPU.
- **Прелоадер** закрывает экран, пока грузятся первые `INITIAL_BATCH = 5`
  (с прогрессом), затем уезжает. Есть таймаут-предохранитель.

Константы — вверху `src/feed.ts`.

**Локальная разработка:** sibling-HTML плейблов не существует на localhost,
поэтому передай `?base=https://<render-host>/`, чтобы iframe'ы указывали на
задеплоенные бандлы, пока крутишь оболочку ленты локально.
